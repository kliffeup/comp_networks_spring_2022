Задание 1 (Wireshark).
1. Заголовок содержит 4 поля: порт отправителя (53512), порт получателя (443), 
   длину заголовка + полезной нагрузки пакета (41 байт) и контрольная сумма (0x9088).
2. Каждый из 4 упомянутых поля имеет длину 2 байта.
3. Как и было отмечено в пункте 1, это длина заголовка UDP (8 байт) + полезная нагрузка этого пакета (33 байта).
4. На длину может быть выделено 2 байта, что может максимально закодировать длину в 65535 байт, из которых 8 байт - это всегда 
   длина собственно заголовка; таким образом максимальная полезная нагрузка UDP-пакета - 65527 байт.
5. Аналогично, с предыдущим пунктом, максимально 2 байта могут закодировать число, равное 65535.
6. Номер протокола UDP -  17 в десятичной системе, 0x11 - в шеснадцатиричной системе.
7. Для первоначального пакета порт отправителя - 53512, порт получателя - 443; для ответного пакета, соотвественно, наоборот: 
   порт отправителя - 443, порт получателя - 53512, таким образом, они просто меняются местами соответственно между собой.

Задание 2 (теоретические задачи).
Пусть L = 1500 * 8 = 12 * 10^3 бит, R = 10^9 бит/с., RTT = 30 мс = 3 * 10^(-2) с.

Хотим, чтобы КПД отправителя было > 0.98.

КПД с использованием окна размера n = n * (L / R) / (RTT + L / R) = (12 * n) / (10^6 * (3 * 10^(-2) + 12 * 10^(-6)) ) = 
(12 * n) / (30000 + 12) = n * 12 / 30012 > 0.98 => n > 0.98 * 30012 / 12 = 2450.98.

Таким образом, размер окна n должен быть >= 2451.
